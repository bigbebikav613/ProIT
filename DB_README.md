# Хранение данных админ-панели в SQL

Добавлен файл базы:
- [proit_admin_data.sql](/C:/Users/bigbebikav/Desktop/лендинг/proit_admin_data.sql)

Что внутри:
- структура таблиц для всего контента лендинга;
- таблица `applications` для заявок;
- минимальные стартовые данные (seed).

## Как использовать
1. Развернуть SQL в SQLite/PostgreSQL/MySQL (с адаптацией синтаксиса при необходимости).
2. Поднять backend API, который читает/пишет эти таблицы.
3. Во фронтенде заменить `localStorage` в `app.js` и `admin.js` на запросы к API.

Важно: статический браузерный фронтенд не может напрямую «записывать в .sql файл», поэтому нужен backend-слой.
